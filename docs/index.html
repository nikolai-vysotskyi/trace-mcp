001 / What It Is

A code graph,
built once, used by every AI agent.

trace-mcp indexes your project into a directed graph — symbols, edges, framework-specific connections — then exposes it through 138 MCP tools. Your AI agent stops reading files and starts traversing structure.

Same code. Two views.

An AI agent reading UserController.php sees a class. trace-mcp sees the route that hits it, the form request that validates input, the Eloquent model it touches, the Inertia page it renders, the Vue component tree, the props each component expects.

That's not a search trick. It's structural understanding from parsing 58 framework integrations — the same connections a senior developer keeps in their head.

Ask anything. Get the graph.

Eight common questions, eight tools that answer them in one call — with the right scope, the right context, and a fraction of the tokens.

004 / How It Works

Three passes. One graph.

trace-mcp builds the graph like a compiler — tree-sitter for parsing, framework plugins for semantics, optional LSP for compiler-grade call resolution.

01

Per-file extraction

tree-sitter parses each file into symbols. Framework plugins detect routes, components, migrations, events, models, schemas.

tree-sitter 81 languages 58 plugins
02

Cross-file resolution

Resolver walks PSR-4, ES modules, Python imports, Vue components, Inertia bridges, Blade inheritance, ORM relations — all into a unified directed edge graph.

imports orm relations framework edges
03

LSP enrichment

Optional pass uses tsserver, pyright, gopls, rust-analyzer for compiler-grade call resolution with 4-tier confidence (resolved / inferred / typed / textual).

tsserver pyright gopls opt-in

009 / Install

Three commands. No configuration.

No API keys. No paid services. Embeddings ship with the binary — runs fully offline after first install.

01

/ Install Globally

npm install -g trace-mcp

Single binary. No native deps. Runs on Node 20+.

02

/ Initialize Once

trace-mcp init

Configures your MCP client (Claude Code, Cursor, Windsurf), installs the guard hook, updates CLAUDE.md.

/ Add A Project

trace-mcp add

Detects frameworks, indexes the codebase, registers the project. Run in each repo you want indexed.

010 / Faq

Common questions.

Do I need API keys or a cloud account?

+
No. trace-mcp is fully local. The default semantic search uses a bundled ONNX model (~23 MB, downloaded once on first use). For optional LLM summarization you can plug in Ollama or OpenAI — but everything else runs without external services.

Where does the index live?

+
In ~/.trace-mcp/ — one SQLite database per project, plus shared decision and topology databases. Nothing is written into your project directory unless you opt in with .traceignore or .trace-mcp/.config.json.

Does it work for monorepos and multi-service codebases?

+
Yes. subproject_add_repo links separate repositories into a unified topology. Cross-service impact analysis traces API calls across service boundaries with confidence scores.

How does it stay up-to-date?

+
A file watcher (300ms debounce) reindexes changed files incrementally. Content-hashed — unchanged files are skipped. The graph is always current without manual reindex.

What about my framework? Is it supported?

+
81 languages and 58 framework integrations ship out of the box. For unsupported frameworks, the plugin API is open — write a custom plugin in TypeScript, register it via ~/.trace-mcp/plugins/. Most plugins are ~200–500 lines.

Is it safe to run on production codebases?

+
All MCP tools are read-only by default. Refactoring tools (apply_rename, apply_codemod) require explicit confirmation and support dry-run preview. The guard hook blocks accidental destructive operations from your AI agent.
