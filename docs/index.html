001 / Problem

AI agents don't understand real codebases.

Large repositories are unstructured, framework logic is implicit, and agents rely on shallow search inside a shrinking context window. The cost shows up as broken changes, silent failures, and hours of manual reconstruction.

01 Large repositories are unstructured and hard to navigate.
02 Framework logic is implicit and inconsistent across languages.
03 Agents rely on shallow search and shrinking context windows.
04 Changes break systems because dependencies aren't understood.

Agents guess instead of reasoning.

002 / Solution

Give agents structured understanding of your code.

trace-mcp reads your code as a system, not as text. One continuously-updated graph, exposed through MCP — every agent, every editor, every session.

From files to functions to call graphs, everything is connected.

+ Build a dependency graph across your entire codebase.
+ Capture real relationships between routes, controllers, models, migrations, components.
+ Understand frameworks and conventions across 58 integrations.
+ Enable safe, context-aware changes with reversible, graph-aware refactors.
003 / Product View

A structured model of your codebase that agents can use.

An AI agent reading UserController.php sees a class. trace-mcp sees the route that hits it, the form request that validates input, the model it touches, the page it renders, the components rendered inside.

This is how agents move from guessing to reasoning.

Global dependency graph — every symbol, every edge, across 81 languages in one directed graph.

Framework-aware parsing — routes to controllers, Inertia to Vue, Eloquent to migrations. 58 integrations, not just syntax.

Real execution relationships — optional LSP enrichment with tsserver, pyright, gopls for compiler-grade call resolution.

Continuously updated — 300ms file-watcher debounce, content-hashed incremental reindex.

004 / How It Works

Three steps. Setup in minutes.

Connect. Index. Expose. After that, agents operate with full context — and the graph stays in sync automatically as you code.

01

Connect your repository

Run trace-mcp add in your repo. Frameworks are detected automatically — Laravel, Django, Next.js, NestJS, Rails, 58 in total.

cli auto-detect no api keys
02

Index into a graph

Tree-sitter parses symbols, plugins resolve framework edges, optional LSP enriches with compiler-grade call data. Stored locally in SQLite + FTS5.

tree-sitter 138 mcp tools local sqlite
03

Expose to your agents

138 MCP tools become available in Claude Code, Cursor, Windsurf, VS Code — or any custom agent that speaks MCP over stdio or HTTP.

mcp stdio + http read-only default

014 / Faq

Common questions.

Do I need API keys or a cloud account?

+
No. trace-mcp is fully local. The default semantic search uses a bundled ONNX model (~23 MB, downloaded once on first use). For optional LLM summarization you can plug in Ollama or OpenAI — but everything else runs without external services.

Where does the index live?

+
In ~/.trace-mcp/ — one SQLite database per project, plus shared decision and topology databases. Nothing is written into your project directory unless you opt in with .traceignore or .trace-mcp/.config.json.

Does it work for monorepos and multi-service codebases?

+
Yes. subproject_add_repo links separate repositories into a unified topology. Cross-service impact analysis traces API calls across service boundaries with confidence scores.

How does it stay up-to-date?

+
A file watcher (300ms debounce) reindexes changed files incrementally. Content-hashed — unchanged files are skipped. The graph is always current without manual reindex.

What about my framework? Is it supported?

+
81 languages and 58 framework integrations ship out of the box. For unsupported frameworks, the plugin API is open — write a custom plugin in TypeScript, register it via ~/.trace-mcp/plugins/. Most plugins are ~200–500 lines.

Is it safe to run on production codebases?

+
All MCP tools are read-only by default. Refactoring tools (apply_rename, apply_codemod) require explicit confirmation and support dry-run preview. The guard hook blocks accidental destructive operations from your AI agent.
