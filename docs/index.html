The problem

AI agents are framework-blind

They can parse syntax, but they can't see that a PHP controller renders a Vue page, or that a model method maps to a database migration three files away.

✗ Without trace-mcp

  • Agent reads 15+ files to understand one feature
  • Doesn't know which Vue page a controller renders
  • "What breaks if I change this?" — agent guesses
  • Schema? Needs a running database
  • Prop mismatch between PHP and Vue? Found in production
  • Burns 600K+ tokens per exploration session

✓ With trace-mcp

  • get_task_context — optimal code subgraph in one shot
  • routes_to → renders_component → uses_prop edges
  • get_change_impact traverses reverse deps across languages
  • Migrations parsed — schema reconstructed from code
  • Prop mismatch detected at index time
  • 94% fewer tokens — ~624K saved per session
Capabilities

Three things no other tool does

Graph traversal + framework awareness + decision memory = structural understanding, not just text search.

Framework-aware edges

Understands that Inertia::render('Users/Show') connects PHP to Vue. 58 integrations across 15 frameworks, 7 ORMs, 13 UI libraries.

Code-linked decision memory

"Chose PostgreSQL for JSONB" is linked to Pool#class. When someone runs get_change_impact, the decision surfaces automatically.

Cross-session intelligence

Past sessions are mined for decisions and indexed for search. get_wake_up gives orientation in ~300 tokens.

Request flow tracing

URL → Route → Middleware → Controller → Service → View, across 15 backend frameworks. One call.

Security scanning

OWASP Top-10 pattern scanning and taint analysis — traces untrusted data from source to sink across function boundaries.

Automated refactoring

Rename symbols across files, extract functions, move code between modules, apply codemods — all graph-aware, all reversible.

Get started

Three commands. That's it.

No configuration files, no API keys, no build steps. Install globally, init once, add your project.

1

Install

npm install -g trace-mcp

2

Initialize

trace-mcp init — configures your MCP client, installs guard hook, adds tool routing to CLAUDE.md

3

Add a project

trace-mcp add — detects frameworks, creates the index. Run in each project directory.

terminal
# Install globally $ npm install -g trace-mcp # One-time setup (MCP clients, hooks, CLAUDE.md) $ trace-mcp init ✓ Configured Claude Code ✓ Installed guard hook ✓ Updated ~/.claude/CLAUDE.md # Register your project $ cd ~/my-app $ trace-mcp add ✓ Detected: Laravel, Vue, Inertia, Tailwind ✓ Indexed 1,247 files, 8,392 symbols ✓ Ready! # Start using in your AI agent > get_task_context("fix the login bug")
